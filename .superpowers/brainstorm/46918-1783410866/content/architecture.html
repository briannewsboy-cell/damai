抢票监控脚本架构

GitHub Actions + Python，监控大麦网并推送邮件/微信

系统数据流
GitHub Actions
每 5 分钟触发一次 cron
↓
Python 主控脚本
  • 检查当前是否在 9:00–20:00
  • 根据日期选择模式：单次检查 / 5 分钟轮询
  • 调用「检查器」获取售票状态
↓
HTTP 检查器
requests + 解析页面
Playwright 检查器
HTTP 被封时启用
↓
大麦网
  • 搜索「刘宪华 苏州 演唱会」
  • 匹配演出详情页
  • 读取按钮状态：即将开售 / 立即购买
↓ 检测到状态变为「立即购买」
邮件通知
SMTP
微信通知
Server 酱 / PushPlus

状态持久化

用仓库内一个 state.json 文件记录上一次检测到的状态，避免重复推送。GitHub Actions 每次运行前读取、运行后提交。
