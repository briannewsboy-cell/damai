数据流与运行时流程

一次 GitHub Actions 运行内部的完整流程

单次运行流程
1
GitHub Actions 触发 — cron 每 5 分钟拉起 workflow
2
读取 state.json — 获取上一次检测状态
3
时间窗口判断 — 不在 9:00–20:00 直接结束
4
日期决定模式 — 7/20 前单次检查；7/20 起进入 5 分钟轮询
5
调用 Damai 检查器 — 搜索关键词 → 匹配演出 → 读取状态
6
状态对比 — 与 state.json 比较，判断是否变化
7
发送通知 — 仅在变为「立即购买」时推送邮件 + 微信
8
写入并提交 state.json — 更新状态，结束运行

状态转换规则

未开售
→ 状态变化 →
立即购买
→ 发通知 →
已通知

如果连续检查都保持「立即购买」，不再重复通知，避免信息轰炸。
